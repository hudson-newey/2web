If you type in the input box, you should see that the value is reflected into a h1 element.

This reactivity is not done through the 2web reactivity system but uses the fact that elements with an "id" attribute creates a variable on the global scope to refer to the element.

You'll see that even though we're compiling this page with the 2web compiler, there is no overhead, and we have not injected any content or JavaScript that you didn't write.

This demonstrates that using 2web is still very useful without the reactivity use-cases:

  1. You can use document partials, so you don't have to write out html boilerplate
  2. You can use the "element ref" shorthand
  3. You get to use the 2web ssg module system
  4. You get an extremely fast compiler
